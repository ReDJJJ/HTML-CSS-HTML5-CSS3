行内块模式 视觉上是底对齐 , 行的高度 由高度最大的元素决定 改动高度最大元素时 会影响其他元素距离顶部的距离
并且每个元素之间还有一定的横向间距4px留白 导致行内块模式十分难做横向布局 以前的横向布局都是在各个a标签行高一致的情况下才能顺利完成
行内元素: 设置widht和height无作用 padding和margin有作用 之间拥有横向间距4px留白 对于行内元素 设置padding 和margin 只会原地散发(文本不动) 并且只有左右边距会与左右元素拉开距离 上下随着内边距padding延伸 会压到上下其他元素 上下外边距margin 没有用 可以利用行高 来设置垂直距离line-height 并且多个并列的文本 会一起对齐往下移动 基线对齐 float模式 共同浮动的元素置于一层排列,无默认间距,且顶部对齐,每个元素之间相互独立,适用于横向布局 横向布局-浮动 总结:文本垂直 -line-height 行高
